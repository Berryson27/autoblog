# 01. 크롤링 로직 (`src/1.crawling.js`)

클래스 `CoupangCrawler` — 쿠팡 상품 페이지에서 **리뷰 텍스트 + 상품 이미지**를 긁어온다.

## 구현 방식

### 1) 브라우저 실행 — `initializeBrowser()`
- **시스템에 설치된 Chrome을 직접 구동**한다 (번들 Chromium 아님).
  - Windows: `C:\Program Files\Google\Chrome\Application\chrome.exe`
- **`headless: false`** — 실제 창을 띄움 (헤드리스 아님).
- Chrome 프로필을 `userData/chrome` 에 영속화 (`--user-data-dir`).
- 봇 탐지 회피 플래그:
  - `--disable-blink-features=AutomationControlled`
  - `ignoreDefaultArgs: ['--enable-automation']`
  - `--disable-web-security`, `--no-sandbox`, `--start-maximized` 등

### 2) 작업 선택 — `run()`
- `99.data/data.json` 읽음 → `crawling !== '1' && productInfo` 인 행만 큐로 추출.
- 처리할 게 없으면 즉시 종료. 있으면 브라우저 1개 띄우고 순차 처리.
- 각 상품 사이 **3초 대기**.

### 3) 상품 1건 크롤링 — `crawlProduct()`
순서:
1. `page.goto(상품URL)` (networkidle2, 60s 타임아웃) — **최대 3회 재시도**(지수 백오프).
2. 팝업(`.popup-modal__content`) 있으면 닫기.
3. `.sdp-review` 리뷰 섹션으로 스크롤.
4. **리뷰 더보기 버튼(`.sdp-review__more-btn`)을 최대 20회 클릭**해 리뷰 로드 (2초 간격).
5. 리뷰 추출 — 두 가지 HTML 구조 모두 대응:
   - 신규: `article` 태그 + `.twc-break-all span[translate="no"]`
   - 구버전 fallback: `.sdp-review__article__list__review__content`
6. 상품 이미지 추출 — 페이지의 **`<script type="application/ld+json">`** 에서 `@type=="Product"`의 `image` 배열을 파싱(JSON-LD 방식, DOM 셀렉터보다 안정적).
7. 이미지를 `node-fetch`로 다운로드 → `99.data/1.postdata/<상품>/image1.jpg ...`
8. 리뷰를 `data.json`(상품 폴더 내)에 `{ reviews: [...] }` 로 저장.
9. **리뷰 0개면 실패 처리**(throw) → 재시도.
10. 성공 시 전역 `data.json`의 해당 행 `crawling = '1'` 로 마킹.

## 입출력 요약
| 입력 | 출력 |
|------|------|
| `99.data/data.json` (상품 URL 목록) | `1.postdata/<상품>/data.json` (리뷰) |
| 쿠팡 상품 페이지 | `1.postdata/<상품>/image*.jpg` (상품 이미지) |
| | `data.json` 의 `crawling="1"` 플래그 |

## 특징 / 취약점
- HTML 구조 변경(쿠팡 리뷰 클래스명 변경)에 **셀렉터가 하드코딩**되어 깨지기 쉬움 → 신규/구버전 이중 대응 + 대체 셀렉터로 방어.
- 봇 탐지 회피는 하지만 `puppeteer-extra-stealth`는 이 파일에선 직접 사용 안 함(기본 puppeteer + 플래그 위주).
- 리뷰 더보기 20회 = 대략 상위 수백 개 리뷰만 수집.
