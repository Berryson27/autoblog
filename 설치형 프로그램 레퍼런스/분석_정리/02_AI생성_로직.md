# 02. AI 글 생성 로직 (`src/2.AI.gemini.js`)

진입 함수 `runAIGrok()` — 크롤링된 리뷰를 **Google Gemini**로 블로그 글(JSON)로 변환하고, 네이버 에디터용 **블록 구조(post.json)** 로 가공한다. (파일명은 gemini지만 함수명은 grok — 이전 버전 흔적)

## 구현 방식

### 1) 작업 선택 — `runAIGrok()`
- `99.data/data.json` 에서 `crawling === '1' && posting !== '1'` 인 행만 처리.
- 각 상품 폴더의 리뷰(`1.postdata/<상품>/data.json`)와 이미지 목록을 읽음.

### 2) API 키 관리 — `loadGoogleApiKeys()`
- `99.data/api.json` 의 `GOOGLE_API_KEYS`(콤마 구분)에서 **여러 개의 Gemini API 키**를 로드.
- 로그에는 마스킹(`앞8자***뒤4자`)해서만 출력.

### 3) 글 생성 — `generateBlogPost()`
- 키 배열을 **Fisher-Yates 셔플** → 부하 분산.
- 모델: **`gemini-flash-latest`**.
- 프롬프트: `99.data/prompt.json` 의 `blogPostPrompt` 템플릿에
  `{productName}`, `{reviewContent}` 플레이스홀더를 치환해 사용.
- **키별 재시도 없음**: 한 키가 실패(할당량초과/무효/네트워크/JSON오류)하면 **즉시 다음 키로 전환**. 모든 키 실패 시 throw.
- 응답에서 ```json 펜스 제거 → 첫 `{` ~ 마지막 `}` 추출 → 트레일링 콤마 정리 → `JSON.parse`.
- 필수 필드 검증: `title`, `content`, `tags`.

### 4) 네이버 블록 변환 — `savePostJson()` (핵심 가공)
AI가 준 `content`(마크다운 비슷한 텍스트)를 **네이버 스마트에디터 블록 배열**로 변환:

| AI 출력 마커 | 변환 결과 |
|--------------|-----------|
| `## 제목` | heading 스타일 text 블록 + 앞에 이미지/구분선 |
| `[HIGHLIGHT] ...` | `quotation`(인용구) 블록 |
| `[KEY] ...` (구버전) | `quotation` 블록 |
| 25자 이하 짧은 문장 | `quotation` |
| 일반 문장 / 번호목록 / "추천·고려" 섹션 | `text` |

- 이미지 삽입 규칙: 섹션마다 상품 이미지를 끼워 넣되 "장점/단점" 섹션은 제외, "결론 및 요약"엔 강제 삽입.
- 광고 처리:
  - `post.json` 설정의 `alllink`(true면 모든 이미지에 상품 URL 링크), `linkbutton`(2면 중앙에도 `button1.png` 버튼) 반영.
  - `0.ad/` 폴더의 광고파일 + 기본 배너(`coupang1.jpg`, `button1.png`) 복사.
  - 최하단에 항상 `button1.png`(링크) + 상품 URL 텍스트 추가.
- 최종 산출물: `2.created_AI_POST/<상품>/post.json` = `{ title, tags, blocks[] }`.
- 처리 후 전역 `data.json` 의 해당 행 `posting = '1'` 마킹(`updateStatusInJson`).

## 입출력 요약
| 입력 | 출력 |
|------|------|
| `1.postdata/<상품>/data.json` (리뷰) | `2.created_AI_POST/<상품>/post.json` (블록) |
| `api.json` (Gemini 키들) | 상품 이미지/광고 이미지 복사본 |
| `prompt.json` (프롬프트) | `data.json` 의 `posting="1"` 플래그 |

## 특징
- **이 단계는 브라우저 불필요** — 순수 REST API 호출 + 파일 I/O. → 헤드리스/백그라운드 100% 가능 (04 문서 참조).
- 키 다수 보유 → 무료 할당량 회전(rotation) 전략.
- 프롬프트가 외부 파일이라 사용자가 글 톤/구조를 커스터마이즈 가능.
