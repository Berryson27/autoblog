# 03. 포스팅 로직 (`src/3.posting.js`)

클래스 `NaverBlogAutoPost` — AI가 만든 `post.json` 블록을 **네이버 블로그 스마트에디터(SE3)** 에 그대로 자동 입력·발행한다. 전체 소스 중 가장 큼(약 1,068줄).

## 구현 방식

### 1) DB/보안 — `initializeDatabase()`, `checkBlockedUser()`
- MongoDB 연결, 스키마 정의: `PostData`(발행이력), `Uploadedata`(글+이미지 백업), `BlockedUser`(차단목록).
- 발행 전 `user.json` 의 `blog-id`를 MongoDB `BlockedUser`와 대조 → **차단 사용자면 작업 중단**. (오류 시 안전하게 차단 처리)

### 2) 브라우저 + 로그인 — `initialize()`
- `1.crawling.js`와 동일하게 **시스템 Chrome + `headless: false` + 영속 프로필**.
- 추가 봇 회피 주입(`evaluateOnNewDocument`):
  - `navigator.webdriver = false`, `window.chrome = {runtime:{}}`, 알림 권한 패치
  - `document.hidden=false`, `visibilityState='visible'` (백그라운드 탭 감지 무력화)
- `https://blog.naver.com/<blogId>?Redirect=Write&` 이동 → `#mainFrame` iframe 로딩 확인.
- **로그인 안 돼 있으면 "수동 로그인 대기"** — 즉, 최초 1회 사람이 네이버 로그인을 해서 프로필에 쿠키를 심어야 함.

### 3) 에디터 진입 — `switchToEditorFrame()`
- `#mainFrame` iframe 컨텐츠로 진입.
- "작성 중인 글" 복원 팝업 뜨면 취소 클릭.
- 제목 placeholder("제목")가 보일 때까지 최대 5회 대기.

### 4) 본문 작성 — `writePost()` + 다수의 헬퍼
`post.json`의 `blocks[]`를 순회하며 블록 타입별로 처리:

| 블록 type | 처리 메서드 | 방식 |
|-----------|-------------|------|
| `text` | `pasteText` | 폰트크기/폰트/색/정렬/굵기 툴바 조작 후 **ClipboardEvent('paste')** 로 텍스트 주입 |
| `image` | `pasteImage` | 로컬 jpg를 base64→Blob→File→DataTransfer→**가짜 paste 이벤트**로 업로드. 이어서 `addLinkToImage`로 이미지에 상품 링크 부착 |
| `quotation` | `insertQuotation` | 인용구 버튼 클릭 후 텍스트 paste, 직후 1x1 투명 GIF(`insertSpacerImage`) 삽입 |
| `divider` | `insertDivider` | 구분선 버튼 클릭 |

- 스타일 적용은 실제 **에디터 툴바 버튼을 클릭**하는 방식(`setFontSize/Family/Color`, `toggleBold`, `setAlignment`) — 셀렉터 하드코딩.
- 태그 문자열 paste, 이어서 `ad.json` 광고 블록을 동일 방식으로 추가.

### 5) 발행 — `clickPublishButton()`
- 사진 라이브러리 요소 클릭 → `publish_btn` → `confirm_btn` 순차 클릭.
- 발행된 URL 획득 → MongoDB `PostData`에 (구글계정/IP/주소/usertype) 기록.

### 6) 배치 루프 — `processJsonAndPost()`
- `data.json`에서 `crawling="1" && posting="1" && upload 비어있음` 인 행만 처리.
- **상품 1건마다 브라우저를 새로 띄우고 끄는 방식**(매 건 `initialize`→발행→`close`).
- 한 건 끝나면 `post.json`의 `postWait`(기본 60분)만큼 **대기 후 다음 글** → 도배 방지/자연스러운 발행 간격.
- 성공 시 `data.json` 행에 `upload='1'`, `uploadAddress=URL` 기록 + MongoDB `Uploadedata`에 글/이미지 백업.

## 입출력 요약
| 입력 | 출력 |
|------|------|
| `2.created_AI_POST/<상품>/post.json` (블록) | 네이버 블로그에 발행된 글 |
| `user.json` (blog-id) | `data.json` 의 `upload="1"`, `uploadAddress` |
| 네이버 로그인 쿠키(Chrome 프로필) | MongoDB `PostData` / `Uploadedata` 기록 |

## 특징 / 핵심 제약
- **클립보드 paste 이벤트 위조**가 입력의 핵심 트릭 — 네이버 에디터의 정상 붙여넣기 흐름을 흉내 내 텍스트·이미지를 주입.
- **반드시 로그인된 Chrome 프로필 필요** (자동 로그인 코드는 없음, 수동 로그인 대기).
- 발행 간격(기본 60분) 때문에 다건 처리 시 **장시간 실행**됨.
- 셀렉터가 네이버 SE3 DOM에 강하게 의존 → 네이버 UI 개편 시 깨짐.
