# 06. 상품 검색 · 보조 기능 (나머지 소스 전체)

핵심 3대 로직(크롤링/AI/포스팅) 외 나머지 기능 파일들을 모두 정리한다.

---

## 0.api_save_json.js (295줄) — 쿠팡 파트너스 API 상품 검색
**파이프라인의 맨 앞 단계.** 크롤링할 상품 URL을 `data.json`에 채워 넣는 역할.

- `loadApiKeys()`: `user.json`에서 쿠팡 파트너스 `accessKey`, `secretKey`, `blog-id` 로드.
- `generateHMAC()`: 쿠팡 오픈API용 **HMAC-SHA256 서명** 생성(`CEA algorithm=HmacSHA256...`). 파트너스 API 인증 필수 로직.
- `searchProducts(keyword, ...)`: 키워드로 상품 검색 (`/products/search`).
- `searchCategoryProducts(categoryId, ...)`: 카테고리 베스트 상품 검색 (`/products/bestcategories`).
- `convertToShortUrl()`: 원본 URL → **파트너스 단축(딥링크) URL** 변환 (`subId: 'cousting'`). → 수익 추적용 제휴 링크.
- `saveToJson()`: 검색 결과를 `data.json`에 행으로 추가. 행 형식:
  `{ productInfo: pageKey, crawling:"", posting:"", upload:"", address: 단축URL, productTitle }`. **중복 productInfo는 스킵.**
- 호출 위치: `app.js`의 `/search-products`, `/save-products` API.

> 즉 **00 개요의 파이프라인 맨 앞**: 검색→`data.json` 적재 → 이후 크롤링/AI/포스팅이 플래그 따라 진행.

## 0.new_save_json.js (103줄) — 단일 URL 수동 추가(CLI)
- `node 0.new_save_json.js "<쿠팡URL>"` 형태로 실행하는 보조 스크립트.
- Puppeteer로 해당 URL 열어 페이지 제목을 가져오고, `productInfo`를 **현재 시각(HHmmss)** 으로 만들어 `data.json`에 1건 추가.
- API 검색 대신 사용자가 링크를 직접 넣고 싶을 때 쓰는 경로. `headless: false`.

## 9.ads_youtube.js (284줄) — 유튜브 광고 자동 시청(조회수/광고 봇)
메인 파이프라인과 **무관한 별도 기능.** 유튜브 영상을 자동 재생한다.
- MongoDB `youtube` 컬렉션에서 `status:'active'`인 영상 URL 목록을 가져옴(`getActiveYoutubeUrls`).
- 시스템 Chrome을 `headless:false`, 작은 창(300x200), `--mute-audio`로 띄움.
- 각 영상 탭마다 **랜덤 User-Agent** 지정, (있으면) 환경변수의 유튜브 로그인 쿠키(SID/HSID/SSID/__Secure-3PSID) 주입.
- `googleads.g.doubleclick.net`, `generate_204` 요청은 **차단(abort)** → 광고 트래킹 회피.
- 영상 자동 `video.play()`, **5분마다 모든 탭 새로고침**(setInterval)하며 무한 시청.
- 브라우저 창 닫으면 **Electron 앱 전체 종료**.
- (선택) `--proxy-server` 환경변수로 프록시 지원.

> ⚠️ 자동 조회수/시청 시간 부풀리기 성격 → 유튜브 정책 위반 소지가 큰 기능.

## postmanagement.js (369줄) — 블로그 검색순위 모니터링
- `dealbbang`이라는 블로그의 발행 글 제목을 긁어옴(`headless:'new'`, 최대 30탭 동시).
- **네이버 검색 OpenAPI**(`openapi.naver.com/v1/search/blog.json`)로 각 제목을 검색해, 해당 블로그 글이 **검색결과 몇 위**인지(또는 "누락") 집계.
- 결과를 `postmanagement.json`에 저장. 자체 Express 서버(포트 3000) + `/search-blog` API.
- 즉 **내 블로그 글의 네이버 검색 노출 순위 추적** 도구.
- 참고: `NAVER_CLIENT_ID/SECRET` 기본값이 코드에 하드코딩(샘플키).

## up_postdata.js (267줄) — 발행 데이터 백업/업로드 라우트
- `Uploadedata` 컬렉션(글 JSON + XML + 이미지 Buffer)을 다루는 Express 라우터.
- `/list-posts` 등으로 로컬 `document/*.json` 글 목록을 조회하고 MongoDB에 업로드/관리.
- 03 포스팅의 `Uploadedata` 백업과 연결되는 보조 관리 기능.

---

## yesmylord.js (404줄) — 관리자(어드민) 백오피스 ★중요
앱 제작자가 사용자를 통제하는 **관리자 전용 라우트 모음**. (이름 "yes my lord" = 어드민)
- `/admin-login`: `ADMIN_PASS` 환경변수로 관리자 세션 로그인.
- **사용자 관리**: 목록 조회, `status` 변경(정상/차단), **유료 만료일(paidExpiryDate) 설정** → 구독/과금 통제.
- **광고 관리**: 광고(이미지/텍스트) 추가·삭제·상태·순서 관리. 이미지는 MongoDB에 Buffer로 저장. 이 광고가 03 포스팅 시 글에 삽입됨.
- **차단 사용자 관리**: `BlockedUser`에 blog-id 추가/삭제 → 03 포스팅의 `checkBlockedUser()`가 이걸 보고 작업 차단.
- 포스트 데이터 조회(페이지네이션).

> 이 파일이 **05번에서 말한 라이선스/차단 체계의 관리자 측**이다. 사용자 PC의 앱은 매 실행마다 이 어드민이 설정한 차단/유료상태를 MongoDB로 확인한다.

## 99.open_chrome.js (31줄) / 99.open_chrome_test.js — 개발용 참고 스크립트
- 주석에 "참고파일"이라고 명시. `chrome-paths`로 크롬 경로 찾아 특정 프로필로 URL 여는 실험용 코드. 실제 파이프라인에서 직접 쓰이지 않음.

## db.js (32줄) — MongoDB 연결 헬퍼
- `connectToMongoDB()`: `MONGODB_URI`/`DB_NAME`으로 mongoose 연결. 실패 시 `process.exit(1)`.

## log.js / log.html / start_ad.js — 로그 뷰어 / 광고시작 트리거
- `log.html`+`log.js`: 앱 UI의 실시간 로그 콘솔(app.js의 logEmitter와 SSE/스트림 연동).
- `start_ad.js`: 9.ads_youtube의 `startAdChrome` 실행 트리거(소형).
