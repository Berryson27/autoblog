# 00. cousting 전체 구조 개요

> 대상: `cousting Setup 2.0.2.exe` 내부 Electron 앱
> 소스 위치: `extracted_cousting\resources\app\`

## 한 줄 정의
**쿠팡 상품 페이지 → 리뷰/이미지 크롤링 → Gemini AI로 블로그 글 자동 생성 → 네이버 블로그에 자동 발행**하는 쿠팡 파트너스(제휴마케팅) 자동화 데스크톱 툴.

## 아키텍처

```
┌─────────────────────────────────────────────────────────┐
│ Electron 메인 프로세스 (main.js)                          │
│  - .env.enc 복호화 → 환경변수 로드 (실패 시 앱 종료)        │
│  - Express 서버(app.js)를 같은 프로세스에서 require        │
│  - BrowserWindow 생성 → http://localhost:3005 로드 (UI)   │
└───────────────────────────┬─────────────────────────────┘
                            │  HTTP (localhost:3005)
┌───────────────────────────▼─────────────────────────────┐
│ Express 서버 (app.js, 포트 3005)                          │
│  - 웹 UI 제공(public/*.html) + REST API                   │
│  - Google OAuth 로그인 / 라이선스(유료기간·차단) 검사       │
│  - POST /run-program {choice} 로 3대 작업 트리거           │
│        choice=2 → 크롤링  / choice=3 → AI / choice=1 → 포스팅│
└───────┬─────────────────┬───────────────────┬────────────┘
        │ choice=2        │ choice=3          │ choice=1
┌───────▼──────┐  ┌───────▼────────┐  ┌───────▼──────────┐
│1.crawling.js │→ │ 2.AI.gemini.js │→ │  3.posting.js     │
│ 쿠팡 크롤러   │  │ Gemini 글 생성  │  │ 네이버 블로그 발행 │
│ (Puppeteer)  │  │ (REST API)     │  │ (Puppeteer)       │
└──────────────┘  └────────────────┘  └───────────────────┘
        │                 │                    │
        ▼                 ▼                    ▼
   data.json 의 상태 플래그로 단계 연결 (파이프라인)
   crawling="1" → posting="1"(=AI생성완료) → upload="1"(발행완료)
   + MongoDB(사용자/발행이력/업로드데이터) + 로컬 이미지/JSON
```

## 데이터 파이프라인 (핵심 — `99.data\data.json`)
모든 단계는 **하나의 `data.json` 배열**을 공유하며, 각 행(상품)의 상태 플래그로 진행을 추적한다.

| 플래그 | 의미 | 설정 주체 |
|--------|------|-----------|
| `productInfo` | 상품 식별자(폴더명) | 사용자/검색 |
| `address` | 쿠팡 상품 URL | 사용자/검색 |
| `crawling = "1"` | 크롤링 완료 | `1.crawling.js` |
| `posting = "1"` | AI 글 생성 완료 | `2.AI.gemini.js` |
| `upload = "1"` | 블로그 발행 완료 | `3.posting.js` |
| `uploadAddress` | 발행된 글 URL | `3.posting.js` |

→ 각 단계는 "직전 단계 플래그가 켜졌고 자기 플래그는 안 켜진" 행만 골라 처리하는 **필터 기반 큐** 방식.

## 디렉터리 (런타임 — 빌드 시 userData 경로)
```
99.data/
├── data.json              ← 작업 큐 (상태 플래그)
├── api.json               ← Google API 키 목록(콤마구분)
├── user.json              ← blog-id 등 사용자 설정
├── post.json              ← 발행 간격(postWait), 링크옵션
├── prompt.json            ← Gemini 프롬프트 템플릿
├── 0.ad/                  ← 광고 블록 원본
├── 1.postdata/<상품>/     ← 크롤링 결과(data.json=리뷰, image*.jpg)
└── 2.created_AI_POST/<상품>/ ← AI 생성결과(post.json, ad.json, 이미지)
public/images/            ← coupang1.jpg, button1.png (기본 배너)
```
