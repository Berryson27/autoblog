# 04. 백그라운드 실행 가능성 분석

> 질문: "이거 백그라운드로 가능하겠어?"
> 결론 한 줄: **단계마다 다르다. AI 단계는 완전 가능, 크롤링/포스팅은 "보이지 않게"는 가능하지만 "완전 무인 자동"은 의도적으로 막혀 있고 위험하다.**

---

## 단계별 판정

| 단계 | 백그라운드(창 숨김) | 완전 무인 자동화 | 비고 |
|------|:---:|:---:|------|
| 크롤링 (1) | △ 가능(개조 필요) | △ | `headless:false` 하드코딩, 탐지 위험 |
| AI 생성 (2) | ✅ 완전 가능 | ✅ | 브라우저 없음, 순수 API |
| 포스팅 (3) | △ 가능(개조 필요) | ✗ | 수동 로그인 전제 + 발행간격 + 차단검사 |

---

## 왜 그대로는 백그라운드가 아닌가

현재 코드는 **명백히 "화면에 띄워서" 돌아가도록** 설계돼 있다:

1. **`headless: false` 하드코딩** (크롤링·포스팅 양쪽)
   - 실제 Chrome 창이 떠야 동작. 창을 닫거나 최소화하면 일부 렌더링/이벤트가 멈출 수 있음.
   - 대응: `document.hidden=false` 강제 주입까지 해둔 걸 보면, 백그라운드 탭에서도 멈추지 않게 애쓴 흔적은 있음.

2. **시스템에 설치된 Chrome을 직접 사용** + **영속 프로필**
   - 번들 Chromium이 아니라 사용자 PC의 Chrome 경로 고정.
   - 그 PC에 Chrome이 켜져 있으면 프로필 충돌 가능.

3. **네이버는 수동 로그인 전제** (`3.posting.js`)
   - 자동 로그인 코드 없음. "로그인 필요. 수동 로그인 대기" 후 진행.
   - 즉 최초 1회는 사람이 붙어 네이버에 로그인해야 함 → 완전 무인 부팅 불가.

4. **GUI/HTTP 트리거 구조**
   - 작업은 Electron 창(localhost:3005 UI)에서 `POST /run-program` 버튼으로 시작.
   - 자체 스케줄러(cron/setInterval) 없음 → 알아서 주기적으로 도는 기능 자체가 없음.

5. **라이선스·차단 검사**
   - 실행마다 MongoDB로 사용자 유료기간/차단 여부 확인. 서버 죽거나 차단되면 멈춤.

---

## "백그라운드로 돌리고 싶다"를 목적별로

### A. 창을 안 보이게 하고 싶다 (화면 가림)
- **부분 가능.** Puppeteer를 `headless: 'new'` 로 바꾸면 창 없이 구동.
  - 단, 쿠팡/네이버 모두 **헤드리스를 더 잘 탐지**한다. 캡차·차단 위험↑.
  - 또는 Windows 작업스케줄러 + 가상 데스크톱/별도 세션으로 창은 띄우되 안 보이게 하는 우회도 있음.

### B. PC를 안 켜놔도(서버에서) 돌리고 싶다
- **크롤링/포스팅: 비현실적.** GUI 데스크톱 앱이고 로그인 세션·시스템 Chrome에 묶여 있어 헤드리스 서버에 그대로 못 올림.
- **AI 단계만**이라면 `runAIGrok()`은 Node + API라 서버에서 단독 실행 가능.

### C. 사람 없이 24시간 자동으로 (무인 운영)
- **불가에 가깝다.** 이유: ①수동 네이버 로그인 전제, ②자체 스케줄러 없음, ③봇 탐지/캡차, ④`postWait`(기본 60분) 발행 간격으로 매우 느림, ⑤계정 차단/라이선스 검사.

---

## 만약 직접 백그라운드화한다면 (참고)
연습 목적의 개념 정리일 뿐, 실제 운영은 약관 위반·계정 정지 위험이 큼.

1. `1.crawling.js`, `3.posting.js` 의 `puppeteer.launch({ headless: false ... })` → `headless: 'new'`.
2. 네이버 로그인: 한 번 로그인한 Chrome 프로필을 그대로 재사용(`--user-data-dir`)하면 쿠키 유지되어 매번 수동 로그인은 피할 수 있음. (단 세션 만료 시 다시 필요)
3. 트리거: 외부에서 주기적으로 `POST http://localhost:3005/run-program` 호출(스케줄러) — 단 Electron 창 띄운 상태 전제.
4. AI만 분리: `node src/2.AI.gemini.js` 형태로 서버에서 단독 배치 실행(파일에 `require.main` 진입점 존재).

---

## 최종 답
- **"화면 가리고 돌리기"**: 코드 한두 줄(headless) 고치면 기술적으로 됨. 단 탐지·차단 위험.
- **"PC 끄고 서버에서 무인 자동"**: 크롤링·포스팅은 사실상 안 됨(GUI+수동로그인+시스템Chrome 의존). AI 글 생성만 따로 떼면 가능.
- 애초에 이 앱은 **사람이 PC 앞에서 창 띄워 놓고 반자동으로 운용**하라고 만든 구조다.
