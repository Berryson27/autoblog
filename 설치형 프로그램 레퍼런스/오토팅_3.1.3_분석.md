## 0. 한 줄 정의

**키워드 검색 → 쿠팡/블로그 크롤링 → Gemini AI로 글·이미지 생성 → 네이버 블로그에 자동 입력·발행(예약발행 포함)까지** 무인으로 도는 제휴마케팅 자동화 데스크톱 봇.

cousting과의 핵심 차이: **완전자동 루프(`5.full_auto.js`)와 예약발행 기능이 새로 추가**됐고, 텍스트 주입이 클립보드 → 실제 키보드 타이핑으로 바뀌었으며, AI 이미지 생성과 캡차 자동 해결, 봇 회피(stealth)가 강화됨.

## 3. 데이터 파이프라인

cousting과 동일하게 `99.data/data.json` 의 **상태 플래그 큐** 방식을 계승.
- `crawling="1"` → 크롤링 완료
- `posting="1"` → AI 글 생성 완료
- `upload="1"` → 발행 완료 / `upload="2"` → 중지로 포기됨(다음 실행 제외)
- `uploadAddress` → 발행된 글 URL

완전자동(`5.full_auto.js`)은 여기에 더해 `99.data/00.fullauto/`(검색이력), `99.data/full_auto.json`(완전자동 설정)을 사용.

## 4. AI 글·이미지 생성 (`src/3.AI.gemini.js`)

### 4.1 글 생성 — 구조화 JSON
- **모델**: `generateBlogPost_model` = `gemini-flash-lite-latest` (`3.AI.gemini.js:341`)
- Gemini에 **responseSchema(JSON 스키마)** 를 강제해 출력 받음. 블록 타입 enum은 `['text','quotation']` (`:723`).
- 출력 구조: `{ title, quotation, contentBlocks[], contentStyles, tags, titleImagePrompt, imagePrompts[] }` (`:861, :731`)
- **다중 API 키 회전**: 키 배열을 Fisher-Yates 셔플 후 한 키 실패 시 즉시 다음 키로 전환(무료 할당량 분산).
- 후처리: 중복 quotation 제거(`:1030`), 본문이 quotation으로 끝나면 잘라서 text로 마무리(`:1050`).

### 4.2 이미지 생성 — 별도 호출 (cousting엔 없던 기능)
- **모델**: `generateImage_model` = `gemini-2.5-flash-image` (`3.AI.gemini.js:340`)
- 글 생성이 만든 `titleImagePrompt`(대표 이미지) + `imagePrompts[]`(본문 이미지 여러 장) 프롬프트로 **AI가 이미지를 직접 생성**. 이미지 생성 전용 API 키도 따로 로드(`:425`).
- → **AI 호출 횟수: 글 1회 + 이미지 N회**(대표 1 + 본문 여러 장). 우리 프로젝트(이미지 수동 업로드, 글 1회)와 가장 큰 차이.

### 4.3 블록 조립 — `createPostJson()`
- AI의 `text/quotation` 블록 사이에 이미지 블록을 끼워넣어 최종 `blocks[]` 구성.
- 광고: 이미지에 상품 링크 부착, 광고 배너·버튼 삽입 옵션.
- 산출물: `2.created_AI_POST/<상품>/post.json` = `{ title, tags, blocks[] }`.

## 5. 포스팅·발행 로직 (`src/4.posting.js`) — 클래스 `NaverBlogAutoPost`

### 5.1 브라우저 실행 (`initialize`)
- **시스템 크롬 사용**: `executablePath` win32 경로 고정 (`:376`)
- **`headless: false`** — 실제 창이 떠야 동작 (`:463`)
- **영속 프로필**: `userDataDir = baseDir/chrome`, `--user-data-dir` 인자 (`:213, :391`) → 한 번 로그인하면 쿠키 유지
- `puppeteer-extra-stealth`로 봇 탐지 회피

### 5.2 로그인 — 수동 1회 전제
- `blog.naver.com/<id>?Redirect=Write&` 이동 후 로그인 상태 확인 (`:582`)
- **로그인 안 돼 있으면 "로그인 필요. 수동 로그인 대기 중..." 후 대기** (`:585-587`). **자동 로그인 코드 없음.**
- 캡차 페이지 감지 시 **`checkAndSolveCaptcha()` 자동 해결 시도** (`:569`) — cousting엔 없던 부분.

### 5.3 본문 작성 — `writePost()` (블록 디스패치 `:1248-1282`)

| 블록 type | 처리 메서드 | 방식 |
|-----------|-------------|------|
| `text` | `typeText` (`:846`) | 툴바로 스타일 적용 후 **실제 키보드 타이핑** `keyboard.type(text,{delay:100})` → Enter 1회 |
| `image` | `pasteImage` (`:1070`) | base64→Blob→File→DataTransfer로 **가짜 `paste` 이벤트** 발생. 이어 "AI 활용 설정" 체크, 링크 있으면 부착 |
| `quotation` | `insertQuotation` (`:907`) | 인용구 버튼 클릭. **6종 스타일** data-value 매핑 (default/line/bubble/underline/postit/corner, `:910-917`). 마지막 컴포넌트 아래 마우스 클릭으로 커서 잡고 입력 |
| `divider` | `insertDivider` (`:1030`) | 구분선 버튼 클릭 |

- 본문 전체 스타일(`contentStyles`): italic·underline·취소선·배경색·줄간격 (`:1238-1245`)
- `pasteText`(클립보드 paste 이벤트, `:826`)도 존재하나 실제 본문은 `typeText`(키보드) 사용
- 1x1 투명 PNG 삽입(`pasteTransparentPixel`, `:870`)로 간격 보정
- 하단에 광고 블록(`ad[]`) 동일 방식으로 추가 (`:1284`)

### 5.4 예약발행 (cousting엔 없던 핵심 신규 기능)
- 네이버 예약발행 UI 자동 조작: 날짜 input(`:1557`), 시간 select "예약 발행 시간 선택"(`:1561`), 분 select(`:1565`)
- 날짜 강제 적용·검증 로직 (`:1797~1999`) — 목표 날짜/시간 설정 후 실제 적용값 검증, 실패 시 재시도
- `post.json` 설정 기준으로 예약 시각 연산, **다음 글 예약 시각을 미리 계산해 연속 예약** (`:2053~2114`)
- 예약발행 ON이면 글 간 대기시간을 생략하고 1분만 대기 (`:2378, :2511`)

### 5.5 발행 버튼 클릭 (`:2031~2230`)
- `button[class*="publish_btn"]` → 설정 적용 → `button[data-testid="seOnePublishBtn"]` / `confirm_btn`
- frame·page·XPath·evaluate 등 **여러 방법으로 끈질기게 자동 클릭** (`:2138~2207`)
- 발행 URL 획득 후 MongoDB 기록

### 5.6 배치 처리 (`processJsonAndPost` 류)
- `data.json`에서 발행 대기 행만 골라 처리. `upload` 플래그로 진행 추적.
- 일반 모드: 글 간 `waitMin~waitMax` 랜덤 대기(도배 방지). 예약발행 모드: 대기 생략.
- 중지 시 시작 안 한 대기 글은 `upload="2"`(포기)로 마킹.

## 6. 완전자동 매니저 (`src/5.full_auto.js`) — 클래스 `FullAutoManager` (신규)

- `start()`에서 **`while(this.running)` 루프**로 키워드별 연속 처리 (`:480, :501`)
- 흐름: 키워드 검색 → 크롤링 → `runBlogPostGeneration`(AI) → `NaverBlogAutoPost.run`(발행)
- 키워드 간 **예약 시각을 보존해 연속 예약** (`nextScheduledPublishAt`, `followingScheduledPublishAt`, `:69-72`)
- 안전장치: 손상 JSON 복구(`extractFirstJsonObject`, `safeParseJson`), 원자적 파일 쓰기(`writeJsonAtomic`), 로그 100개 버퍼
- 트리거: 사용자가 UI에서 "시작"을 눌러야 동작. **시계 기반 자동 시작(cron) 기능은 없음** — 무인 부팅은 외부 스케줄러 필요.

### 예약발행 설정 (`app.js`)
- `/get-scheduled-settings` (`app.js:1022~`): `scheduledFirstTime`(기본 180분 뒤 첫 발행), `scheduledIntervalMin`(기본 60), `scheduledIntervalMax`(기본 120)
- `scheduledPost: on/off` 프리미엄 설정 (`:982, :1010`)

### 요약
- **우리 것**: 가볍고 단순. 이미지 올리고 AI 글 만들어 네이버 에디터에 블록 자동 입력까지. 발행·크롤링·예약·완전자동·서버/DB·과금은 **전부 없음** = 사람이 검토 후 직접 발행하는 반자동 작성 도구.
- **오토팅**: 거기에 크롤링·AI 이미지 생성·자동발행·예약발행·완전자동 루프·캡차해결·MongoDB·라이선스·프록시까지 갖춘 상용 자동화 봇.
- 겹치는 핵심(Electron / AI 글 생성 / 네이버 블록 입력)은 동일. 오토팅이 **수집·발행·운영 레이어를 통째로 더** 갖고 있는 구조.

## 9. 핵심 제약/리스크 (오토팅 기준)

- `headless:false` — 창이 떠 있어야 함. 창 닫으면 멈출 수 있음.
- 네이버 SE3 DOM 셀렉터 하드코딩 → 네이버 UI 개편 시 깨짐.
- 캡차/세션 만료 시 사람 개입 필요(자동해결은 100% 보장 아님).
- 자동발행·크롤링은 약관 위반·계정 정지 리스크 존재.
- 시계 기반 무인 시작 불가(외부 작업 스케줄러로 앱 실행 자체를 걸어야 함).
