# 네이버 블로그 풍부한 글쓰기 가이드 (AI 작성 기준)

> 목적: AI가 글을 **"마커 기계적으로 박기"가 아니라, 글의 역할을 이해해 네이버 표현을 골라 쓰도록** 하는 편집 기준.
> 이 문서는 나중에 생성 프롬프트(generationService)와 파서(draftModel)에 녹일 **표준 스펙**이다.
> 핵심 사상 한 줄: **"모든 문장에는 역할이 있고, 역할에 맞는 표현을 고른다. 강조는 절제할수록 강해진다."**

---

## 1. 네이버가 제공하는 표현 수단 (인벤토리)

### 1-1. 블록/글감 (툴바 1열)
| 수단 | 용도 | 블로그 가치 |
|---|---|---|
| 사진 | 이미지 | ★★★ 필수 |
| 인용구(6종) | 핵심 문장 강조 박스 | ★★★ |
| 구분선 | 큰 단락 경계 | ★★ |
| 링크 | 외부/내부 연결 | ★★ |
| 표 | 비교·스펙·가격 정리 | ★★ |
| 소스코드 | 코드 블록 | ★(기술글) |
| 장소 | 지도/업체 카드 | ★★(방문후기) |
| 글감(책·영화·뉴스) | 콘텐츠 카드 | ★(리뷰글) |
| 동영상·파일·일정·수식·스티커·MYBOX·내돈내산 | 특수 용도 | 상황별 |

### 1-2. 텍스트 서식 (툴바 2열)
- **문단 스타일**: 대제목 / 소제목 / 인용 / 본문 → 글의 **뼈대(제목 계층)**
- **글꼴·크기**: 강약 조절 (큰 글씨=강조, 작은 글씨=부연)
- **굵게 / 기울임 / 밑줄 / 취소선**: 단어·구절 단위 강조
- **글자색 / 배경(형광펜)**: 키워드 하이라이트
- **정렬 / 줄간격 / 리스트**: 가독성·리듬
- **위·아래첨자 / 특수문자**: 보조 표기

---

## 2. 핵심 원칙 (3가지)

1. **역할 일치**: 표현은 "예뻐서"가 아니라 **그 문장이 하는 일**에 맞춰 고른다.
2. **절제**: 강조 수단은 적게 쓸수록 강해진다. 다 강조하면 = 아무것도 강조 안 한 것.
3. **리듬(대비)**: 본문-강조-이미지가 번갈아 나와야 눈이 안 지친다. 같은 형태가 연속되면 단조롭다.

---

## 3. 표현별 사용 기준 (Do / Don't)

### 인용구 (quotation)
- **언제**: 그 섹션에서 **딱 하나의 핵심 주장 / 감정의 정점 / 기억에 남길 한 문장**.
- **분량**: 섹션당 **0~1개**, 글 전체 **2~4개**. 짧고 강하게(한 문장).
- **Don't**: 평범한 설명문을 인용구로 넣기, 연속 2개 이상, 긴 문단 통째로.
- **스타일 선택(6종)**: 기본=범용 / 라인=깔끔한 강조 / 말풍선=대화·후기 / 포스트잇=팁·메모 / 밑줄=핵심어 / 모서리=마무리 멘트.

### 소제목 (heading) — *권장 추가 항목*
- **언제**: 글을 2~5개 섹션으로 나눌 때 각 섹션 머리.
- **효과**: 스캔 가독성↑, 목차감. 인용구보다 **구조 잡는 데 우선**.

### 강조 — 굵게 / 색 / 형광펜
- **언제**: 문단 안에서 **진짜 중요한 키워드 1~2개**만.
- **분량**: 문단당 1~2곳. 색은 글 전체에서 **1~2색**으로 통일.
- **Don't**: 한 문장에 여러 색, 문장 전체 굵게, 무지개 색.

### 작게 / 덜 강조 (small·gray) — *권장 추가 항목*
- **언제**: 부연 설명, 캡션, 출처, 주의·면책("가격은 변동될 수 있어요") 같은 **곁다리 정보**.
- **효과**: 본문과 위계를 만들어 중요한 게 더 도드라짐.

### 리스트 (번호·불릿) — *권장 추가 항목*
- **언제**: 3개 이상 나열(준비물, 순서, 장단점).
- **Don't**: 문장 하나를 억지로 리스트로.

### 표 (table) — *권장 추가 항목*
- **언제**: 항목×속성 비교(가격표, 사양, 메뉴), 2열 이상 데이터.

### 구분선 (divider)
- **언제**: **큰 주제 전환**에만(도입→본론, 본론→마무리). 보통 글당 1~3개.
- **Don't**: 문단마다.

### 이미지 (image)
- **언제**: 섹션마다 1장 정도, 글 흐름에 맞춰. 텍스트와 **교차** 배치.
- **캡션**: 사진 아래 짧은 설명은 **작게** 처리하면 좋음.

### 링크 (link)
- **언제**: 근거·출처·관련 글. 본문 흐름 방해 안 되게.

## 4. 글 전체 구조와 리듬 (모델)

```
[제목]                          ← 핵심 키워드 포함
도입 문단(본문)                  ← 흥미/공감, 1~2문단
  └ 대표 이미지
── 구분선(선택) ──
[소제목 1]                       ← 섹션 머리
  본문 + (키워드 강조 1~2)
  └ 이미지
  [인용구] 이 섹션의 핵심 한 줄    ← 0~1개
[소제목 2]
  본문 …
  (작게) 부연/주의               ← 곁다리는 작게
  └ 이미지 / 표(필요시)
── 구분선 ──
마무리 문단(본문)                ← 요약 + 행동유도
  [인용구] 여운 남기는 한 줄(선택)
#태그 …
```

**리듬 규칙(요약):**
- 인용구: 섹션당 0~1, 전체 2~4
- 강조색/굵게: 문단당 1~2곳, 색은 1~2종 통일
- 이미지: 섹션당 ~1장, 텍스트와 교차
- 구분선: 주제 전환에만(1~3개)
- 문단은 2~4문장, 문단 사이 한 줄 여백

---

## 5. 콘텐츠 의도 → 표현 → 마커 매핑표 (실전)

| 글에서의 역할/의도 | 골라야 할 네이버 표현 | 우리 마커 / 현재 지원 |
|---|---|---|
| 섹션의 핵심·정점 한 문장 | 인용구 | `[QUOTE]` (스타일 `[QUOTE2]`~`[QUOTE6]`) ✅ |
| 섹션 머리(구조) | 소제목 | ❌ 미지원 → **추가 필요** |
| 키워드 강조 | 굵게/색 | (통로 있음) ⚠️ 문장단위 지정 안 됨 |
| 부연·캡션·주의 | 작은 글씨/회색 | ❌ 미지원 → **추가 필요** |
| 3개+ 나열 | 리스트 | ❌ 미지원 → **추가 필요** |
| 비교·스펙·가격 | 표 | ❌ 미지원 |
| 큰 주제 전환 | 구분선 | `---` ✅ |
| 시각 자료 | 이미지(+캡션) | `<img src="image_N">` ✅ (캡션은 일반 텍스트) |
| 근거·출처 | 링크 | 이미지 링크만 ✅ / 텍스트 링크 ❌ |

---

## 6. 안티패턴 (하지 말 것)
- ❌ **인용구 남발**: 문장 셋 중 하나가 인용구 → 강조가 죽는다.
- ❌ **전부 강조**: 굵게·색 떡칠 → 가독성 최악.
- ❌ **무지개 색**: 한 글에 색 3종 이상.
- ❌ **구분선 떡칠**: 문단마다 줄.
- ❌ **의미 없는 표/리스트**: 1~2개짜리 억지 표.
- ❌ **역할 무시**: 평범한 설명을 인용구·큰 글씨로.

---

## 7. AI에게 줄 작업 절차 (프롬프트화 핵심)
AI가 글을 "쓰고 끝"이 아니라 **2단계**로 일하게 한다:
1. **초안 작성** → 글의 논리/섹션 구성.
2. **편집 패스**: 각 문단의 **역할을 판단**해서 표현 부여 —
   - 이 섹션 핵심 문장 1개 → 인용구
   - 섹션 머리 → 소제목
   - 키워드 1~2개 → 강조
   - 곁다리 정보 → 작게
   - 나열/비교 → 리스트/표
   - 주제 전환 → 구분선 + 이미지
   - 단, **2~4절제 한도**를 지킨다.

---

## 8. 우리 도구 현황 vs 추가하면 좋은 것
- **현재 지원**: 텍스트(키보드 타이핑), 이미지(+캡션·링크), 인용구 6종, 구분선, 태그. per-block 스타일 통로(크기·색·정렬·굵기)는 있으나 AI가 안 채움.
- **현재 마커**: `[QUOTE]` / `[QUOTE2]~[QUOTE6]` / `---` / `<img src="image_N">`
- **가성비 높은 추가 후보(우선순위)**:
  1. **문장단위 강조**(굵게/색) — 인프라 있음, 프롬프트+마커만 → 효과 큼
  2. **소제목(heading)** — 구조 가독성 핵심
  3. **작게/부연 스타일** — 위계 표현
  4. **리스트**, 그다음 **표**

---

## 한 줄 결론
네이버의 표현은 많지만, 핵심은 **"글의 역할을 이해 → 맞는 표현을 절제해서 → 리듬 있게"**. 이 문서의 §3 기준과 §4 리듬, §5 매핑을 프롬프트에 녹이면, AI가 단순 삽입이 아니라 **편집된 글**을 만들 수 있다.
