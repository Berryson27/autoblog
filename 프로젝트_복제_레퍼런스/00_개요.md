# 프로젝트 복제 레퍼런스

## 목적

이 문서는 현재 `autoclick` 프로젝트를 다른 프로젝트에서 참고해 비슷한 앱을 만들기 위한 레퍼런스다.

핵심은 코드 복사가 아니라 구조와 흐름 재사용이다.

## 프로젝트 성격

- Electron 데스크톱 앱
- Renderer 화면에서 사용자가 이미지, 기획값, 블로그 ID를 입력
- Main process가 IPC 요청을 받아 파일, AI 생성, 브라우저 자동화를 처리
- Playwright로 네이버 로그인/글쓰기 화면을 직접 열고 에디터에 초안을 삽입
- AI 생성은 Gemini API 호출 결과를 비동기 작업으로 관리

## 현재 기능 범위

- 로컬 이미지 선택 및 드래그 앤 드롭
- 이미지 순서 변경, 삭제, 캡션 입력
- 글 기획값 입력
- 이미지 기반 블로그 초안 생성 요청
- 생성 결과 제목, 본문, 태그 편집
- 생성 본문 안의 이미지 플레이스홀더를 실제 이미지 블록으로 변환
- 네이버 블로그 글쓰기 화면 열기
- 네이버 에디터에 제목, 텍스트, 이미지, 캡션, 소제목, 인용구, 구분선 삽입
- 최종 결과 텍스트 복사

## 의도적으로 하지 않는 것

- 자동 발행
- CAPTCHA 또는 2FA 우회
- 서버 저장
- Firebase 연동
- 로그인 자동 입력

## 재사용할 때 우선 가져갈 개념

- Electron preload를 통한 제한된 IPC API
- Renderer 상태를 최종 Naver payload로 변환하는 중간 draft model
- 이미지 파일을 path/name/mime/order/caption 형태의 asset으로 정규화
- AI 결과를 사람이 편집 가능한 제목/본문/태그로 분리
- 본문 마커를 Naver 에디터 블록으로 변환
- Playwright persistent context로 로그인 세션 유지
- 네이버 에디터 iframe 내부를 단계별로 조작
