# 재구현 체크리스트

## 1. 앱 골격

- Electron main process 구성
- BrowserWindow 생성
- contextIsolation 활성화
- nodeIntegration 비활성화
- preload에서 필요한 API만 노출
- Renderer는 Node API에 직접 접근하지 않게 구성

## 2. 이미지 입력

- 파일 선택 dialog 제공
- 드래그 앤 드롭 경로 추출
- 지원 확장자 제한
- 파일 존재와 크기 검증
- 중복 제거
- 최대 개수 제한
- caption, order, enabled 상태 유지

## 3. Renderer 상태

- 단계형 workflow 상태
- busy 상태
- 이미지 목록
- 기획 입력값
- 생성 job 상태
- 편집 가능한 최종 제목/본문/태그
- 제출 버튼 활성화 조건

## 4. Draft model

- Renderer 상태를 순수 데이터로 정규화
- AI 원문에서 제목 추출
- 태그 라인 분리
- 이미지 마커를 실제 이미지 block으로 치환
- 미참조 이미지 처리 정책 결정
- 최종 Naver payload 생성
- 복사용 plain text 생성

## 5. AI 생성

- API key 로딩 정책 결정
- 이미지 개수 제한
- 이미지 base64 변환
- prompt 작성
- 비동기 job 관리
- processing/completed/failed 상태 제공
- 실패 메시지 사용자용으로 변환

## 6. Playwright 세션

- persistent context 사용
- userDataDir 고정
- headless false
- Chrome channel 우선 시도
- 실패 시 기본 Chromium 사용
- page crash/error/dialog 로그 연결
- 수동 로그인 전제

## 7. 네이버 글쓰기 이동

- Blog ID 필수 검증
- 글쓰기 URL 생성
- 현재 URL이 글쓰기 화면인지 확인
- 아니면 글쓰기 URL로 이동
- 로그인 상태가 아니면 사용자가 직접 처리하게 둠

## 8. 에디터 자동 입력

- mainFrame iframe 대기
- 임시글 복구 팝업 닫기
- 제목 영역 탐색
- 본문 영역 탐색
- selector 실패 시 좌표 fallback 준비
- 각 삽입 단계에 timeout 적용
- frame detach 발생 시 재시도
- 디버그 로그 남기기

## 9. 블록 삽입

- text: 키보드 입력
- image: paste 이벤트로 File 삽입
- heading: 스타일 적용 후 입력, 기본 스타일 복원
- quotation: toolbar로 삽입 후 quote 영역 탈출
- divider: toolbar 사용, 실패 시 fallback
- image link가 필요하면 마지막 이미지 선택 후 link toolbar 사용

## 10. 테스트

- 태그 정규화
- block 정규화
- 이미지 asset 생성
- AI prompt 구성
- generation job 상태 전환
- AI 응답 parsing
- image marker 매핑
- Naver payload 생성

## 고치면 어떻게 되는지

현재 방식:

- 네이버 화면을 직접 조작한다.
- 구현 난이도는 높지만 실제 에디터 결과를 사람이 보는 화면과 맞출 수 있다.
- 네이버 UI 변경에 취약하다.

API/서버 방식으로 바꾸면:

- 안정성은 좋아질 수 있다.
- 네이버 공식 글쓰기 API가 없거나 제한적이면 같은 결과를 만들기 어렵다.
- 로그인/세션/권한 처리가 별도 문제가 된다.

AI 생성만 분리하면:

- 데스크톱 앱은 가벼워진다.
- API key 노출 정책을 서버에서 관리할 수 있다.
- 대신 서버 운영과 사용자 데이터 전송 이슈가 생긴다.

Playwright를 제거하면:

- CAPTCHA/로그인/UI 변경 리스크는 줄어든다.
- 자동 에디터 입력 기능은 사실상 사라진다.
- 최종 텍스트 복사 도구에 가까워진다.
