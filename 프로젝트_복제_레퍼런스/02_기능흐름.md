# 기능 흐름

## 1. 이미지 수집

사용자는 파일 선택 dialog 또는 드래그 앤 드롭으로 이미지를 추가한다.

Main process는 이미지 경로를 검사해 지원 확장자만 asset으로 만든다.

asset에 포함되는 값:

- id
- path
- name
- mimeType
- order
- caption
- enabled

중복 경로는 제거하고 최대 30개까지만 사용한다.

## 2. 이미지 편집

Renderer는 이미지 목록을 카드 형태로 보여준다.

지원 동작:

- 순서 이동
- 역순 정렬
- 삭제
- 캡션 입력

순서가 바뀌면 `order`를 다시 정규화한다.

## 3. 기획값 입력

사용자는 글 작성 조건을 입력한다.

사용되는 값:

- 작성자 프로필
- 말투
- 타깃 독자
- 주제
- 업체/장소/상품 정보
- SEO 키워드
- 요청사항

이 값들은 AI prompt에 들어간다.

## 4. AI 생성

Renderer가 workflow draft를 만들어 Main에 보낸다.

Generation service는 다음 순서로 처리한다.

- generationId 생성
- jobs Map에 processing 상태 저장
- 백그라운드로 생성 실행
- 이미지 파일을 base64 inline data로 변환
- prompt와 이미지를 Gemini API에 전송
- 성공 시 completed와 aiResponse 저장
- 실패 시 failed와 error 저장

결과 조회는 별도 버튼으로 수행한다.

## 5. 생성 결과 편집

AI 응답에서 제목을 추출한다.

제목 추출 우선순위:

- HTML h1/h2
- Markdown heading
- 첫 번째 유효 라인

본문 끝의 태그 라인은 별도 tags로 분리한다.

사용자는 최종 제목, 본문, 태그를 직접 수정할 수 있다.

## 6. 본문을 블록으로 변환

draft model은 생성 본문을 Naver 블록 배열로 바꾼다.

지원 마커:

- `image_1` 계열: 이미지 블록
- Markdown 소제목: heading 블록
- 인용 마커: quotation 블록
- `---`: divider 블록
- 일반 문단: text 블록

본문에 참조되지 않은 이미지는 뒤에 자동 추가된다.

## 7. 네이버 제출

최종 제출 시 Renderer는 Naver payload를 만든다.

Main process는 다음 순서로 처리한다.

- payload 검증
- 글쓰기 URL 생성
- 현재 Playwright page가 글쓰기 화면인지 확인
- 아니면 글쓰기 URL로 이동
- 에디터 iframe 찾기
- 기존 임시글 복구 팝업 닫기
- 제목 입력
- 본문 영역 포커스
- 블록을 순서대로 삽입

## 8. 에디터 삽입 방식

텍스트:

- 키보드 입력 방식
- 문단 구조는 줄바꿈 횟수를 제한해 정리

이미지:

- 로컬 파일을 읽어 Blob/File로 만든 뒤 paste 이벤트로 삽입
- 삽입 전후 이미지 개수를 비교해 성공 여부 확인

소제목:

- 폰트 크기와 bold를 적용하고 입력 후 기본 스타일로 복원

인용구:

- 네이버 에디터 toolbar의 quotation 버튼 또는 스타일 옵션 사용
- 입력 후 quote 영역 아래를 클릭해 다음 일반 문단으로 빠져나온다

구분선:

- toolbar horizontal line 버튼 사용
- 실패하면 텍스트 구분선으로 fallback
