# 주의사항

## 네이버 에디터 의존성

이 프로젝트는 네이버 SmartEditor의 DOM, iframe, toolbar selector에 강하게 의존한다.

네이버가 UI를 바꾸면 다음 부분이 깨질 수 있다.

- iframe 이름
- 제목/본문 contenteditable selector
- toolbar button selector
- quotation style data-value
- image component selector
- draft restore popup selector

그래서 editor service는 반드시 분리해두는 편이 맞다.

## 로그인 정책

자동 로그인은 하지 않는다.

사용자가 Playwright 브라우저에서 직접 로그인하고, persistent context가 세션을 유지한다.

이 방식이 현실적으로 안전하다. 로그인 자동 입력이나 우회는 유지보수와 정책 리스크가 크다.

## 이미지 삽입

이미지는 파일 input이 아니라 paste 이벤트로 넣는다.

장점:

- 네이버 에디터가 실제 붙여넣기처럼 처리한다.
- 삽입 순서가 editor 흐름과 맞기 쉽다.

주의:

- 대용량 이미지가 많으면 느려질 수 있다.
- 삽입 후 DOM count 확인이 필요하다.

## 텍스트 입력

텍스트는 paste보다 keyboard typing을 기본으로 사용한다.

장점:

- 사람 입력과 비슷하다.
- paste 이벤트로 인한 포맷 꼬임을 줄인다.

단점:

- 긴 글은 느리다.

속도를 올리려면 delay를 낮출 수 있지만, 너무 빠르면 에디터가 입력을 놓칠 수 있다.

## AI 결과 파싱

AI 응답은 항상 기대한 형식으로 오지 않는다.

따라서 다음 fallback이 필요하다.

- 제목이 없으면 첫 줄을 제목으로 사용
- 태그가 없으면 빈 태그 허용
- 없는 image_N 마커는 warning 처리
- 본문에 안 나온 이미지는 뒤에 추가

## 인코딩

현재 일부 한국어 문자열이 깨져 있다.

새 프로젝트에서 복제할 때는 다음을 권장한다.

- 모든 파일 UTF-8 저장
- UI 문구 전면 재작성
- README 재작성
- prompt 문구 재작성

## 테스트 기준

자동화 테스트는 DOM 실제 입력까지 완전히 보장하지 않는다.

테스트로 잡을 수 있는 것:

- 데이터 정규화
- prompt 생성
- job 상태
- payload 변환
- image marker parsing

수동 확인이 필요한 것:

- 네이버 로그인 세션
- 실제 에디터 입력
- 이미지 삽입
- toolbar 스타일 적용
- 글쓰기 화면 UI 변경 여부
